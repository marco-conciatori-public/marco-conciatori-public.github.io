Mantis' Hardware Components

In case you want to build my same robot buying only the necessary components, or you want to create something different, here I list all the packages that I used, specifying which components were used and which were not. But keep in mind, the robot could still grow and change, so this list may be expanded in the future.

From RDK X3 Robot (Yahboom)

RDK X3 Development Board

The main processor. It came already set up with Ubuntu 18.04.

ROS Expansion Board

Features the main switch, connections for wheel and arm motors, and distributes power from the battery to all other connected components.

USB HUB Board

Increases the number and type of ports available for connecting to the RDK X3 Development Board (despite the name, not just USB ports).

RGB Cooling Hat

Includes a fan for cooling the board stack (RDK X3, ROS Expansion, USB HUB) and some lights controllable via the PS2 Controller.

OLED Screen

Used for displaying text, just 4 lines with about 20 characters per line. It is useful for checking the robot's state when a computer connection is not available. It is not strictly necessary, but it is quite useful.

IMX219 MIPI Camera

A camera that connects to the RDK X3 Development Board with a wide, flat ribbon cable. Used for object recognition and streaming to the VR.

MS200 Lidar

Senses the direction and distance of objects in a 2D horizontal plane (if mounted perpendicular to the terrain). It comes with a dedicated adapter board.
Note: In the current setup, the lidar is partially occluded and can only see in front and to the sides because of other components behind it. Mounting it higher with some kind of pole or platform would solve that but make it blind to any obstacle lower than itself.

Wireless PS2-like Controller

Used for manual control of the robot; its behavior can be customized via software. The battery is included.

Battery

Powers the ROS Expansion Board and all connected components. It can last from 30 minutes to a few hours depending on usage. Its specific charger is also included.

Antenna

One antenna for Wi-Fi connection and hotspot function.

External Components

There are also pieces that are auxiliary and not physically part of the robot, like a TFCard-to-USB converter to read/write the TF Card with a computer. Many parts like screws and pillars have some spares.

Unused components

All the chassis parts, the wheels, and the motors from the RDK X3 robot are not part of the current design. The included 64 GB TF Card (system memory) was substituted with a 128 GB one just because I had it, you can keep the original.

From DOFBOT - Jetson Nano (Yahboom)

Robotic Arm

A pre-assembled robotic arm with 6 Degrees of Freedom (DoF).

Jetson Nano Board

The arm's computer, which in the current setup, is used only for voice interaction. It includes a wireless network card.
Note 1: the Yahboom one is configured to boot from USB instead of a TF Card. It is possible to revert the change, everything is explained on their robotic arm and Jetson Nano guides, but I kept it as is.
Note 2: the Jetson Nano, for now, is used only for the voice interaction, so it is not strictly necessary. I am using it mainly because I already have it. However, it is possible in the future it will be used for more functions.

U Disk

A metal USB drive that contains the OS and memory for the Jetson Nano. It can be changed for more memory, the provided one has 64 GB.

Robotic Arm Expansion Board

It is equivalent to the ROS Expansion Board for the Jetson Nano. It has input for battery plus power distribution, power switch, connections for motors, etc...

USB Camera

A 0.3 MP USB camera with a support for attaching it to the top of the robotic arm. Used for voice interaction, so that the reasoning network can "watch" through the camera if that is required to answer user's questions. In theory, you don't need this camera, the IMX219 MIPI Camera from the RDK X3 Robot can be used for this purpose. However, I used it because it was already included with the robotic arm and because it is convenient (the camera is directly connected to the Jetson Nano, where the images are needed, and in this way there are no problems with many component competing for the same resource).

Antenna

Two antennas for Wi-Fi connection, with a supporting piece of metal. Because of their positions, they are one of the most exposed and fragile parts of the whole robot. Note also that they tend to come unscrewed easily.

External Components

As for the RDK X3 Robot, there are external components. For example, there are gadgets for testing the arm AI capabilities: a mat with delimited areas and cubes with images or colors on each side. Not necessary, but it is useful for tests, or as an exercise in controlling the robotic arm.

Unused components

Unused components include:

  • a (second) OLED Screen. It is possible, however, that I or someone else in the future would add the code for it to the current implementation
  • a cabled PS2-like controller
  • a transformer to power up the Robotic Arm Expansion Board from an electric plug
  • the support base for the arm

The controller and transformer are not used because the robot can't have cables to external objects.

From Mechanum Wheel Chassis (Yahboom)

Chassis

A large chassis configured to host the robotic arm, featuring a "pendulous suspension" to ensure all four mechanum wheels maintain ground contact for correct operation.
It already has many screw holes, but I still had to add quite a few to secure every component. I even had to make a larger hole on the bottom plate for the battery cable, but it depends on how you organize the various boards and sensors.

Wheels and Motors

Includes larger mechanum wheels and motors. These motors have a higher reduction ratio, making them slower but capable of carrying more weight.

Separately Acquired Components

Coral USB Accelerator (Google)

It is an Edge TPU, a USB device that a computer can delegate specific computations to. In the current setup it is used to run the YOLO models locally (for object recognition and the autonomous vision agent). From my tests it speeds up the processing of a single camera frame by almost 10 times.
Note: I bought the Coral USB Accelerator before adding the robotic arm (with the attached Jetson Nano). I didn't do any test, but it is possible the Jetson Nano could entirely replace the Coral USB Accelerator.

Battery for the Jetson Nano (Zeee)

A Lipo battery (3S 5200mAh with Deans T Plug) that powers all devices on the Robotic Arm Expansion Board (Jetson Nano, microphone, speakers, USB Camera). It fits on the space I had available, and with 5200mAh probably lasts a lot more than the battery connected to the main processor. Consider I had to also buy an extension cable from the battery to the Robotic Arm Expansion Board, because the battery cable is short.
Note 1: you also need a charger for the battery. I am using WANGCL IMAX B6AC Lipo Battery Charger B6 80W Digital LCD, but it is not specific for this battery, as long as you know what you are doing, any compatible charger is fine.
Note 2: the Robotic Arm Expansion Board and Jetson Nano can't monitor the battery level, and lithium batteries can be damaged if discharged too much. So you may want to monitor the battery level manually or buy a component that can do it for you. Since the battery I chose lasts longer than the main battery, I use that as recharge reminder.

ReSpeaker Mic Array v2.0 (Seeed Studio)

A microphone array with sound filtering capabilities and Direction of Arrival (DoA) computation. Currently, it is connected to the Jetson Nano and for voice interaction (it can distinguish between ambient sounds and voices). To use the Sound Agent (the mode in which the robot follows any sound it hears), the microphone must be connected to the main board.

Speakers

Since the speakers have to be connected to the ReSpeaker Mic Array v2.0, they need a 3.5mm audio jack connection. Aside from that requirement, almost any device small enough to fit on the robot can be used. I had two unused speakers, so I removed their casing and attached them to the robot.

VR Headset

If you want to use the robot with a VR headset, you need one on which you can install/run custom applications. I am using (and only tested it with) Meta Quest 3.

Other Customizations

Various custom parts including

  • 128 GB TF Card, a memory expansion for the RDK X3 Development Board. You can keep the 64 GB one from the RDK X3 robot.
  • The cables necessary to connect all the components are included in both the RDK X3 Robot and the DOFBOT - Jetson Nano. However, I had to buy a few specific cables that adapted physically to the new configuration (some were too short, or needed the head to curve in a different direction).
  • I also added an extension cable to the 40-pin interface of the RDK X3 Development Board because, in the current configuration, adding or removing connections to the pins is Very inconvenient. Even just counting the pins to connect the cables to the correct pins is not an easy task.
  • Since I had various LEDs and switches, I connected them to the 40-pi interface of the RDK X3 Development Board, so that some important functions of the robot can be used even without controller or pc connection. For example, if you want to use the autonomous Vision Agent mode (automatically search for a target and move toward it), a controller should not be needed, it is autonomous after all!
  • For fixing all the components to the chassis you would need extra screws, pillars, and other supporting pieces.
